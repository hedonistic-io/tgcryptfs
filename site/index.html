Features

Military-grade encryption meets unlimited cloud storage

Encryption

Post-Quantum Cryptography

XChaCha20-Poly1305 for symmetric encryption. ML-KEM-768 (FIPS 203) for key exchange. Your data is protected against both classical and quantum adversaries.

Storage

Unlimited Telegram Backend

Files are encrypted, chunked, and stored as messages in your Telegram Saved Messages. No storage limits. No monthly fees. Your data, your account.

Transparency

FUSE Filesystem Mount

Mount encrypted volumes as regular directories. Every application works seamlessly — editors, media players, scripts. No API needed.

Privacy

Opaque Database Schema

SQLite metadata uses BLAKE3-derived table and column names. Stolen databases reveal zero structural information without the schema key.

Sharing

Multi-User Access Control

Share volumes with other users via ML-KEM key wrapping. Granular permissions (Read, Write, Admin). Time-limited invite tokens with revocation.

Safety

Deadman Switch

Configurable triggers that auto-destroy all data: heartbeat timeouts, network checks, OS events. If you can't check in, your data disappears.

Efficiency

Content Deduplication

Content-defined chunking with BLAKE3 hashing enables block-level deduplication. Store the same data once, reference it everywhere.

Recovery

22-Word Sentence Reference

Back up your 256-bit root key as 22 human-readable words. Write them down. Store them in a safe. Recover your entire volume from memory.

Security

Forward Secrecy

Epoch-based key rotation derives fresh encryption keys per epoch. When you rotate, old keys are zeroized. Past data becomes unrecoverable even with the root key.

AAD Binding

Every ciphertext is bound to its context via Additional Authenticated Data. A block encrypted for file 42 at offset 0 cannot be moved to file 43 or offset 1024 — the authentication tag will fail. This prevents replay, reorder, and substitution attacks.

Installation

Three ways to get started, on any platform

Recommended

Install Script

One-line installer for Linux and macOS. Auto-detects your OS and architecture, installs FUSE dependencies, downloads the latest release with SHA-256 verification, and sets up shell completions.

curl -fsSL https://raw.githubusercontent.com/hedonistic-io/tgcryptfs/main/scripts/install.sh | bash
Rust Users

Cargo Install

If you have a Rust toolchain (1.75+), install directly from source via Cargo. Requires FUSE development headers on your system.

cargo install tgcryptfs-cli
Build from Source

Clone and Build

Full control. Clone the repo, build with cargo, run the test suite. A build script checks prerequisites and handles platform-specific setup.

git clone https://github.com/hedonistic-io/tgcryptfs.git
cd tgcryptfs && ./scripts/build.sh --install
